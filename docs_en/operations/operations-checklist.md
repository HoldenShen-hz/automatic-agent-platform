## 2. Pre-Coding Checklist

Source: `pre_coding_checklist.md`

Before starting a new feature or refactoring, must confirm:

### Scope and Design

- [ ] Corresponding issue or ADR exists
- [ ] ADR has been reviewed and accepted (if involving core contract changes)
- [ ] Impact scope has been assessed (affecting which modules, which tests)
- [ ] Backward compatibility strategy exists, or incompatibility has been marked as breaking change
- [ ] Schema change has migration script if applicable

### Code Quality

- [ ] TypeScript strict mode compliant
- [ ] No new `as unknown as` additions (type safety rule)
- [ ] No bare `console.log` (use StructuredLogger)
- [ ] No direct `process.env` (through config loader)
- [ ] Public APIs have type signatures
- [ ] Error codes conform to `error_code_registry.md` specification

### Testing

- [ ] Corresponding unit test file exists
- [ ] Corresponding integration test exists (if cross-module)
- [ ] Tests verified with `npm run test`
