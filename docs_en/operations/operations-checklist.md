## 2. Pre-Coding Checklist

Source: `pre_coding_checklist.md`

Before starting new features or refactoring, must confirm:

### Scope and Design

- [ ] Corresponding issue or ADR exists
- [ ] ADR reviewed and accepted (if core contract changes are involved)
- [ ] Impact scope assessed (which modules, which tests are affected)
- [ ] Backward compatibility strategy exists, or incompatibility marked as breaking change
- [ ] Migration script exists if schema changes are involved

### Code Quality

- [ ] TypeScript strict mode compliant
- [ ] No new `as unknown as` (type safety rules)
- [ ] No bare `console.log` (use StructuredLogger)
- [ ] No direct `process.env` (use config loader)
- [ ] Public APIs have type signatures
- [ ] Error codes comply with `error_code_registry_contract.md` specification

### Testing

- [ ] Corresponding unit test file exists
- [ ] Corresponding integration tests exist (if cross-module)
- [ ] Tests verified via `npm run test`
