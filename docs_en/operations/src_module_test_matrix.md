# Src Module Test Matrix

> This document is automatically generated by `npm run test:matrix` to answer “which unit tests exist for each `src` module, which integration tests exist, and which files still lack direct coverage”.

## 1. Statistics Rules

- Modules are categorized by `src` top-level directory, `src/core/*` subdivided to `core/<sub-module>`.
- `tests/unit/` is considered unit tests; `tests/integration/`, `tests/e2e/`, `tests/golden/` are considered integration tests.
- “Direct coverage” is inferred by test files with the same basename or same directory token, for maintenance perspective, does not replace coverage reports.
- `index.ts` and type-only files also enter the matrix; if there is no direct test, they appear in “lacking direct coverage”.
