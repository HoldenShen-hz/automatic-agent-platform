## 1. 量化交易 (Quantitative Trading)

### 核心 Agent 工作流

- **信号生成 Agent**：摄入行情数据（L1/L2 盘口、Tick、另类数据），运行因子模型（alpha 信号），输出排序后的交易想法。流程：数据摄入 → 特征工程 → 模型推理 → 信号排序 → 风控过滤 → 订单生成。
- **回测 Agent**：接收策略定义，回放历史数据，模拟成交（含滑点/手续费模型），输出绩效报告（Sharpe、最大回撤、Calmar）。
- **执行 Agent**：接收目标组合，计算执行计划（TWAP/VWAP/IS），跨交易所路由订单，监控成交率，实时调整算法参数。
- **风险管理 Agent**：持续监控组合敞口（行业、因子、Greeks），执行仓位限制，触发熔断，计算 VaR/CVaR，生成追保通知。
- **组合优化 Agent**：运行均值方差、Black-Litterman 或风险平价优化，受约束条件限制（换手率、行业上限、ESG 过滤）。

### 关键工具/集成

| 类别     | 具体工具                                                                                     |
| -------- | -------------------------------------------------------------------------------------------- |
| 行情数据 | Bloomberg B-PIPE, Refinitiv Elektron, Wind（万得）, CTP/FEMAS（期货）, IEX Cloud, Polygon.io |
| 交易执行 | FIX 4.2/4.4 网关, 券商 OMS/EMS API（Interactive Brokers, 中信/华泰 PB 系统）, DMA 直连       |
| 回测引擎 | Zipline, Backtrader, QuantConnect, 自研事件驱动引擎                                          |
| 风控     | RiskMetrics, Axioma, Barra 因子模型, 内部 VaR 引擎                                           |
| 基础设施 | KDB+/q 时序数据库, Redis 状态缓存, Kafka 事件流, FPGA/内核旁路（超低延迟）                   |
| 监管报送 | 证监会报送接口, SEC EDGAR, MiFID II 交易报告                                                 |

### 领域风险画像

- **财务损失**：错误信号或执行 Bug 可在数秒内造成灾难性损失（Knight Capital 45分钟亏损4.4亿美元）
- **乌龙指风险**：Agent 下单方向/数量/价格错误
- **模型衰减**：Alpha 信号随市场变化退化，Agent 必须检测 regime change
- **流动性风险**：Agent 假设不存在的流动性，导致巨额滑点
- **系统性风险**：关联 Agent 策略可能放大市场崩盘（闪崩动态）
- **数据投毒**：被污染的行情数据导致错误决策

### 数据敏感度分级

- **极度机密**：交易策略、alpha 信号、持仓数据、盈亏（核心 IP）
- **机密**：历史回测结果、风控参数、客户组合配置
- **内部**：行情数据（通常有许可证限制再分发）、执行分析

### 监管/合规要求

- **中国（证监会）**：算法交易登记、委托/成交比限制、自成交防范、算法策略日报、A 股高频策略限制
- **美国（SEC/FINRA）**：Reg NMS 合规、Reg SHO 卖空规则、CAT 报告、市场操纵（spoofing/layering）检测、SEC Rule 15c3-5 风控要求
- **欧盟（MiFID II）**：算法登记、kill switch、实时监控、T+1 交易报告
- **通用**：盘前风控检查是法律强制的。必须维护每笔订单决策的审计轨迹

### 性能/延迟要求

- **行情处理**：HFT <1ms tick-to-signal；中频 <100ms
- **下单**：个位数微秒（FPGA）到低毫秒级（取决于策略类型）
- **风控检查**：盘前检查不得增加 >50μs 订单路径延迟
- **回测**：必须在分钟内回放数年 Tick 数据（并行化）
- **可用性**：交易时段 99.99%，零容忍错过开市

### HITL 要求

- **强制人工审批**：策略上线生产、风控限额变更、新品种接入、资金分配变更、影响最大仓位的参数变更
- **人工监控**：实时盈亏/敞口/订单流仪表盘。人工必须能随时一键 kill-switch 任何 Agent
- **盘后审核**：所有交易必须可被合规审查。每日盈亏和风险报告签字确认

### 评估指标

- Sharpe Ratio, Sortino Ratio, 最大回撤, Calmar Ratio
- 信息比率（alpha vs benchmark）
- 执行质量：Implementation Shortfall, VWAP 滑点
- 成交率、拒单率、委托/成交比
- 风险调整后收益归因
- 策略容量（多少 AUM 前 alpha 衰减）
- 系统正常运行时间、延迟分位数（p50/p99/p999）

### 常见故障与恢复

| 故障模式           | 恢复策略                                                 |
| ------------------ | -------------------------------------------------------- |
| 数据源损坏/延迟    | 切换备用源、陈旧数据检测、数据缺口 >N秒 自动平仓         |
| 策略产生极端信号   | 盘前合理性检查（最大单量/最大名义金额/速率限制）、熔断器 |
| 执行场所断连       | 自动路由到备用场所、订单排队、通知人工                   |
| 风控限额突破       | 立即平仓、禁用策略、通知风控经理                         |
| 模型在生产中过拟合 | 在线监控信号衰减、自动降低策略权重、regime 检测          |

## 2. 电商 (E-Commerce)

### 核心 Agent 工作流

- **商品上架 Agent**：生成商品描述、优化标题 SEO、商品分类、从图片提取属性、管理多平台上架（天猫/京东/Amazon）
- **定价 Agent**：监控竞品价格、运行动态定价模型（需求弹性、库存水位、利润目标）、实施降价/促销计划
- **库存履约 Agent**：按 SKU/仓库预测需求、触发补货、优化仓库分配、协调三方物流、处理拆包发货
- **客服 Agent**：处理售前咨询（尺码、兼容性）、售后问题（退货、退款、投诉）、升级复杂案例、生成回复模板
- **推荐 Agent**：构建用户画像、运行协同过滤/内容/混合模型、个性化首页/搜索/邮件、A/B 测试推荐策略
- **风控 Agent**：实时交易评分（速率检查、设备指纹、地址不匹配）、标记可疑订单、管理拒付争议

### 关键工具/集成

| 类别      | 具体工具                                                              |
| --------- | --------------------------------------------------------------------- |
| 平台      | Shopify API, 天猫/淘宝开放平台, 京东开普勒, Amazon SP-API, 拼多多     |
| 支付      | 支付宝, 微信支付, Stripe, PayPal, Adyen                               |
| 物流      | 顺丰 API, 菜鸟, FedEx/UPS/DHL API, WMS（Manhattan, Blue Yonder）      |
| 搜索/推荐 | Elasticsearch, Algolia, Pinecone（向量搜索）, TensorFlow Recommenders |
| 分析      | Google Analytics 4, Mixpanel, Amplitude, 自建点击流管线               |
| CRM       | Salesforce, HubSpot, 有赞, 微盟                                       |

### 领域风险画像

- **定价错误**：Agent 将价格设为 ¥0.01 而非 ¥100——在许多司法管辖区具有法律约束力
- **超卖**：承诺不存在的库存，导致取消订单和声誉损害
- **客服幻觉**：Agent 做出不存在的承诺（免费退货、保修）
- **推荐偏见**：信息茧房、推高利润商品而非高质量商品、歧视
- **风控误判**：误拦合法客户会摧毁转化率

### 数据敏感度分级

- **PII（高）**：客户姓名、地址、手机号、支付信息（PCI-DSS 范围）
- **机密**：定价策略、供应商成本、利润数据、库存水位
- **内部**：商品目录、聚合销售数据、A/B 测试结果

### 监管/合规要求

- **中国**：《电子商务法》、《消费者权益保护法》、广告真实性要求、七天无理由退货、数据隐私（PIPL）
- **全球**：PCI-DSS（支付数据）、GDPR（欧盟客户）、CCPA（加州）、产品安全法规、FTC 广告指南
- **平台规则**：各市场有上架政策、禁售品、刷评操纵规则

### 性能/延迟要求

- **搜索/推荐**：p99 <200ms（页面加载贡献）
- **价格更新**：竞争响应准实时（<5分钟），计划促销批量即可
- **风控评分**：每笔交易 <500ms（同步结账流程）
- **客服**：聊天首次响应 <3s，工单 <2h
- **库存同步**：多渠道实时（防止超卖），<1分钟延迟可接受

### HITL 要求

- **强制**：价格变动 >X% 于当前价、超阈值退款、管控品类上架（食品/药品/儿童用品）、升级投诉、促销预算审批
- **审核**：前 N 次客服回复（训练期）、推荐模型重训、风控规则变更

### 评估指标

- GMV、转化率、客单价、购物车放弃率
- 客户满意度（CSAT/NPS）、首次解决率
- 库存周转率、缺货率、库存积压率
- 价格竞争力指数、利润保持率
- 风控精确率/召回率、拒付率
- 推荐 CTR、每次会话收入、多样性/新颖度

### 常见故障与恢复

| 故障模式                 | 恢复策略                                             |
| ------------------------ | ---------------------------------------------------- |
| 价格机器人战（竞相降价） | 底价约束、利润护栏、阈值处人工告警                   |
| 库存同步延迟致超卖       | 基于预留的库存管理、安全库存缓冲、自动补偿受影响客户 |
| 客服 Agent 幻觉政策      | 基于政策检索的接地生成、强制引用政策文件             |
| 推荐冷启动               | 热门商品兜底、人口统计默认值、显式偏好收集           |
| 支付网关故障             | 多 PSP 切换、排队重试、客户通知                      |

## 3. 广告推广 (Advertising & Marketing)

### 核心 Agent 工作流

- **投放规划 Agent**：分析商业目标、历史投放数据和受众数据，提出媒介计划（渠道组合、预算分配、排期、定向条件）
- **创意生成 Agent**：生成适配平台规格的广告文案/图片/视频脚本（抖音15秒竖版视频、微信朋友圈卡片、Google 自适应展示广告），生成 A/B 测试变体
- **竞价优化 Agent**：跨 DSP 管理实时竞价，根据转化概率、预算节奏、时段、竞争态势调整出价，实现目标 CPA/ROAS 策略
- **受众管理 Agent**：从一方/二方/三方数据构建受众分群，管理 Lookalike 扩展、排除列表、频次上限、跨设备身份打通
- **归因与报告 Agent**：收集跨触点转化数据，运行多触点归因模型（Shapley/Markov/数据驱动），生成效果看板和优化建议

### 关键工具/集成

| 类别     | 具体工具                                                                                          |
| -------- | ------------------------------------------------------------------------------------------------- |
| 广告平台 | Google Ads API, Meta Marketing API, 巨量引擎（Ocean Engine/TikTok）, 腾讯广告, 百度营销, 快手磁力 |
| DSP/SSP  | The Trade Desk, DV360, MediaMath                                                                  |
| 创意     | Canva API, Figma API, Midjourney/DALL-E（图片生成）, RunwayML（视频）                             |
| 分析     | Google Analytics, Adobe Analytics, AppsFlyer/Adjust（移动归因）                                   |
| CRM/CDP  | Segment, mParticle, Salesforce CDP, 神策数据                                                      |
| 品牌安全 | IAS, DoubleVerify, MOAT                                                                           |

### 领域风险画像

- **品牌安全**：广告出现在不当内容旁（恐怖主义、仇恨言论、竞品内容）
- **预算浪费**：竞价优化 Agent 在低转化人群过度投入
- **创意合规**：生成的广告违反平台政策（虚假宣传、禁止内容）或商标/版权问题
- **受众隐私**：基于敏感属性定向（健康/宗教/政治）违规
- **点击欺诈**：在机器人流量上浪费预算

### 数据敏感度分级

- **PII（高）**：客户邮件列表、CRM 数据（自定义受众）、设备 ID
- **机密**：投放效果数据、竞价策略、获客成本、创意测试结果
- **内部**：聚合触达/频次数据、市场基准

### 监管/合规要求

- **中国**：《广告法》——严格限制绝对化用语（"最好"）、医疗/健康声明、明星代言、儿童广告；《互联网广告管理办法》广告标识要求
- **全球**：GDPR 跟踪/定向同意（欧盟）、CCPA 退出、Apple ATT 框架、Google Privacy Sandbox；FTC 代言指南
- **平台政策**：各平台有独立创意审核流程和禁止品类

### 性能/延迟要求

- **竞价决策**：RTB 拍卖 <100ms
- **投放调整**：每小时预算节奏、每日出价优化周期
- **创意生成**：文案分钟级、图片/视频小时级（异步工作流）
- **报告**：准实时看板（15分钟延迟可接受）、每日/每周自动报表

### HITL 要求

- **强制**：投放启动审批（预算承诺）、创意上线前审批（品牌/法务审核）、敏感品类受众定向审核、预算增加 >X%
- **建议**：每周效果回顾需人工签字的优化建议、新渠道接入

### 评估指标

- ROAS（广告投资回报率）、CPA（获客成本）、CPC、CPM
- CTR、转化率、浏览转化率
- 触达量、频次、GRP（毛评点）
- 品牌提升（知名度/考虑度/好感度）
- 创意质量分（平台标准）、互动率
- 归因准确性（增量测试）
- 预算节奏准确性（实际 vs 计划支出）

### 常见故障与恢复

| 故障模式                        | 恢复策略                                            |
| ------------------------------- | --------------------------------------------------- |
| 竞价错误致预算超支              | 平台层硬性每日/每小时预算上限、实时支出监控自动暂停 |
| 创意被平台审核拒绝              | 提交前合规检查 Agent、已审核通过格式模板库          |
| 受众过窄无法投放                | 自动受众扩展触发、Lookalike 兜底                    |
| 归因数据丢失（Cookie/追踪问题） | 模型化转化、营销组合模型（MMM）备份                 |
| 广告疲劳（CTR 下降）            | 自动创意轮换、频次上限强制、刷新触发器              |

## 4. 金融服务 (Financial Services)

### 核心 Agent 工作流

- **信贷评估 Agent**：收集申请人数据（财务报表/征信/替代数据），使用评分卡/ML 模型计算信用分，生成审批/拒绝建议及解释，结构化贷款条款
- **KYC/AML Agent**：验证身份证件（OCR+活体检测），筛查制裁名单（OFAC/UN/EU），监控可疑交易模式（拆分/分层/回路），提交可疑交易报告（SAR/STR）
- **保险承保 Agent**：分析风险因子（健康记录/财产数据/理赔历史），定价保单，识别除外责任，生成保单文件
- **理赔处理 Agent**：接收理赔申请（文件/照片/医疗记录），验证保障范围，检测欺诈指标，估算赔付金额，路由至理赔师审核或自动审批低额理赔
- **监管报表 Agent**：跨系统汇总数据，生成监管报表（Basel III 资本充足率/CCAR 压力测试/银保监报表），验证完整性，通过规定渠道提交

### 关键工具/集成

| 类别        | 具体工具                                                                                  |
| ----------- | ----------------------------------------------------------------------------------------- |
| 征信        | 人行征信, Experian, Equifax, TransUnion, 百行征信                                         |
| 制裁/反洗钱 | World-Check（Refinitiv）, Dow Jones Risk & Compliance, OFAC SDN 名单, Chainalysis（加密） |
| 文档处理    | ABBYY, Tesseract OCR, AWS Textract, 自研证件解析器                                        |
| 核心银行    | Temenos, FIS, 长亮科技, 中电金信                                                          |
| 保险平台    | Guidewire, Duck Creek, 中科软                                                             |
| 监管报送    | XBRL 生成器, 银保监 EAST 系统, SEC EDGAR XBRL                                             |

### 领域风险画像

- **模型偏见**：信用评分模型歧视受保护群体（公平借贷违规）
- **AML 漏检**：未能检测洗钱——巨额监管罚款（丹麦银行：20亿美元）
- **可解释性**：监管机构要求模型可解释（美国贷款拒绝通知要求）
- **数据泄露**：金融数据泄露 = 监管处罚 + 集体诉讼 + 声誉毁损
- **系统性风险**：跨机构关联 Agent 决策放大市场压力

### 数据敏感度分级

- **极度敏感（PII+金融）**：身份证号/SSN、银行账号、征信报告、医疗记录（保险）、纳税申报
- **机密**：内部风控模型、定价算法、持仓信息、客户名单
- **受监管**：所有交易数据须遵守留存要求（通常5-7年）

### 监管/合规要求

- **银行**：Basel III/IV 资本要求、Dodd-Frank（美国）、银保监会法规（中国）、PSD2（欧盟）
- **反洗钱**：Bank Secrecy Act（美国）、《反洗钱法》（中国）、欧盟第4/5/6号反洗钱指令
- **保险**：Solvency II（欧盟）、偿二代（中国 C-ROSS）、各州保险法规（美国）
- **数据**：GLBA（美国金融隐私）、PIPL（中国）、GDPR（欧盟）
- **AI 专项**：欧盟 AI Act 将信用评分归为高风险 AI——要求合规评估、人工监督、偏见测试

### 性能/延迟要求

- **实时决策**：欺诈评分 <200ms，信用预审 <5s
- **KYC 验证**：自动检查 <30s，增强尽职调查 <24h
- **理赔处理**：简单理赔自动裁决 <1分钟，复杂理赔含人工 <48h
- **监管报表**：批处理，但必须满足严格截止日期（通常 T+1 或月度）

### HITL 要求

- **强制**：超阈值贷款审批、SAR/STR 报告（法律要求人工审核）、超自动裁决限额的保险理赔、模型部署/重训、任何不利信贷决策（必须可复查）
- **监管**：许多辖区要求对影响消费者的自动化决策有"有意义的人工参与"

### 评估指标

- 信贷：基尼系数、KS 统计量、通过率、违约率、违约损失率
- AML：SAR 质量（监管反馈）、误报率、已知类型检测率
- 保险：赔付率、综合成本率、承保准确性、理赔周期
- 通用：监管检查发现、客户投诉、模型稳定性（PSI）、运营损失事件

### 常见故障与恢复

| 故障模式                       | 恢复策略                                               |
| ------------------------------ | ------------------------------------------------------ |
| 模型漂移导致批准不良信贷       | PSI 监控、冠军-挑战者测试、自动模型回退                |
| AML 误报过载（90%+误报是常态） | 基于风险的优先排序、反馈循环优化规则、分层审查流程     |
| 监管报表数据不一致             | 对账检查、数据血缘追踪、提交前验证规则                 |
| 部署了有偏见的模型             | 部署前公平性测试（差异影响分析）、按受保护群体持续监控 |
| KYC 证件伪造                   | 多因素验证、活体检测、与政府数据库交叉比对             |

## 5. 数据处理 (Data Engineering)

### 核心 Agent 工作流

- **管线编排 Agent**：设计、生成和管理 ETL/ELT 管线，解读自然语言数据需求，生成 DAG（Airflow/Dagster），处理调度、重试和依赖管理
- **数据质量 Agent**：对入站数据做 Profiling，应用校验规则（Schema 一致性/范围检查/唯一性/引用完整性），隔离坏记录，生成质量报告，建议修复
- **Schema 管理 Agent**：检测源系统 Schema 变更，提出迁移脚本，评估下游影响，管理 Schema 注册中心，强制向前/向后兼容
- **数据血缘 Agent**：从源头到消费追踪数据流转，构建血缘图谱，回答影响分析查询（"表 X 变了会影响什么？"），支撑审计追踪
- **异常检测 Agent**：监控数据量、新鲜度、分布漂移和管线延迟，基于根因假设告警

### 关键工具/集成

| 类别      | 具体工具                                                          |
| --------- | ----------------------------------------------------------------- |
| 编排      | Apache Airflow, Dagster, Prefect, dbt, Luigi                      |
| 流处理    | Apache Kafka, Flink, Spark Structured Streaming, Pulsar           |
| 存储      | Snowflake, Databricks, BigQuery, Hive, Delta Lake, Apache Iceberg |
| 质量      | Great Expectations, dbt tests, Monte Carlo, Soda                  |
| 目录/血缘 | Apache Atlas, DataHub, Amundsen, OpenLineage                      |
| Schema    | Confluent Schema Registry, Protobuf, Avro, JSON Schema            |

### 领域风险画像

- **静默数据损坏**：坏数据在管线中未被检测地传播，污染下游分析和 ML 模型
- **管线故障级联**：一个失败任务阻塞跨组织的依赖管线
- **Schema 破坏性变更**：源系统变更在没有警告的情况下中断消费者
- **数据丢失**：错误转换丢弃记录、幂等性失败导致重复
- **成本爆炸**：低效查询或不受控的数据增长导致云预算超支

### 数据敏感度分级

- 按内容变化——Agent 必须具有敏感数据感知能力
- **高**：PII 列（必须脱敏/令牌化）、金融数据、健康数据
- **中**：业务指标、运营数据
- **低**：公开数据集、参考数据
- 数据工程 Agent 本身需要访问元数据，但应最小化对实际敏感数据的访问

### 监管/合规要求

- **数据治理**：数据留存策略、删除权（GDPR 第17条）、数据驻留（数据主权法——中国《数据安全法》要求特定数据留在境内）
- **血缘/审计**：SOX 合规要求金融报告数据血缘；BCBS 239 银行数据汇总要求
- **访问控制**：最小权限原则、敏感字段列级安全

### 性能/延迟要求

- **批处理管线**：SLA 驱动（如每日聚合在早6点前就绪）
- **流处理**：实时场景亚秒级，准实时秒级
- **数据质量检查**：不得增加 >10% 管线运行时间
- **血缘查询**：典型企业数据资产影响分析 <5s
- **Agent 响应**：管线生成/修改秒级；复杂优化分钟级

### HITL 要求

- **强制**：Schema 迁移审批（破坏性变更）、生产管线部署、数据删除请求、敏感数据集访问授权、影响成本的基础设施变更
- **审核**：首次生产运行前审核生成的管线代码、数据质量规则变更、异常处理决策

### 评估指标

- 管线可靠性：SLA 达成率、故障率、平均恢复时间
- 数据质量：质量检查通过率、数据新鲜度、完整性、准确性分数
- 效率：管线运行时间、计算成本、数据存储成本趋势
- 覆盖率：有质量检查的数据集占比、血缘覆盖率、目录覆盖率
- Agent 专项：管线生成正确率、vs 手动开发节省时间

### 常见故障与恢复

| 故障模式               | 恢复策略                                              |
| ---------------------- | ----------------------------------------------------- |
| 管线运行中失败         | 检查点、幂等操作、自动退避重试、告警                  |
| 源 Schema 变更中断管线 | Schema 漂移检测、新增字段自动适配、破坏性变更人工审核 |
| 数据质量回退           | 自动隔离坏批次、回退到最后已知良好数据、SLA 违规告警  |
| 成本失控（计算/存储）  | 预算告警、自动扩缩容限制、执行前查询成本估算          |
| 重试导致数据重复       | 幂等写入模式（upsert/去重键）、尽可能精确一次语义     |

## 6. 代码开发 (Software Development)

### 核心 Agent 工作流

- **代码生成 Agent**：接受自然语言需求或设计文档生成实现代码，理解代码库上下文（导入/约定/现有模式），配合生成测试
- **代码审查 Agent**：分析 PR 中的 Bug、安全漏洞、风格违规、性能问题和架构问题，提供行级注释、解释和修复建议
- **测试 Agent**：生成单元/集成/E2E 测试，识别未测试代码路径，创建测试夹具和 Mock，维护覆盖率目标
- **CI/CD Agent**：管理构建管线，解读构建失败，建议修复，处理部署编排，管理功能开关，执行金丝雀分析
- **安全扫描 Agent**：运行 SAST/DAST/SCA 工具，分类结果，减少误报，建议修复代码，跟踪漏洞生命周期
- **调试 Agent**：分析错误日志/堆栈追踪/指标，假设根因，建议修复，在测试环境复现问题

### 关键工具/集成

| 类别     | 具体工具                                                                  |
| -------- | ------------------------------------------------------------------------- |
| 版本控制 | GitHub, GitLab, Bitbucket API                                             |
| CI/CD    | GitHub Actions, Jenkins, GitLab CI, CircleCI, ArgoCD                      |
| 安全     | Snyk, SonarQube, Semgrep, Trivy, Dependabot, CodeQL                       |
| 测试     | Jest, Pytest, JUnit, Playwright, Cypress, k6（负载）                      |
| 代码分析 | Tree-sitter（AST 解析）, Language Servers（LSP）, linters（ESLint, Ruff） |
| 项目管理 | Jira, Linear, GitHub Issues                                               |
| 监控     | Sentry, Datadog, PagerDuty, Grafana                                       |

### 领域风险画像

- **代码质量退化**：Agent 生成的代码能工作但不可维护，引入技术债
- **安全漏洞**：Agent 引入注入缺陷、硬编码密钥、不安全依赖
- **幻觉 API**：Agent 使用当前库版本中不存在的函数/方法
- **许可证合规**：生成的代码侵犯版权/GPL 许可代码
- **过度依赖**：开发者盲目接受 Agent 建议，降低代码理解

### 数据敏感度分级

- **极度敏感**：源代码（核心 IP）、密钥/凭证（绝不能出现在代码中）、部署配置
- **机密**：构建日志（可能含内部路径/配置）、安全扫描结果、架构图
- **内部**：公开依赖信息、通用编码标准

### 监管/合规要求

- **许可证合规**：必须追踪生成代码来源，避免 GPL 污染专有代码库，遵守 OSS 许可义务
- **安全标准**：SOC 2 Type II（SaaS）、ISO 27001、OWASP Top 10 修复
- **行业特定**：HIPAA（医疗软件）、PCI-DSS（支付处理代码）、DO-178C（航空软件）
- **AI 生成代码**：AI 生成代码披露的新兴法规，版权地位在法律上尚未明确

### 性能/延迟要求

- **代码补全**：内联建议 <500ms（IDE 体验）
- **代码审查**：典型 PR <5分钟（异步可接受）
- **测试生成**：单函数秒级，模块分钟级
- **安全扫描**：增量分钟级，全代码库小时级
- **CI/CD**：构建/测试管线不应被 Agent 瓶颈——并行执行

### HITL 要求

- **强制**：所有生成代码合并前必须经过人工开发者审查；生产部署需人工审批；安全漏洞修复决策；架构决策
- **建议**：Agent 建议的重构执行前审查、依赖升级决策、任何认证/授权代码变更

### 评估指标

- 代码正确性：生成代码测试通过率、Bug 引入率
- 代码质量：可维护性指数、圈复杂度、linter 通过率
- 审查质量：Bug 检测真阳性率、开发者建议采纳率
- 生产力：每任务节省时间、PR 周期缩短、测试覆盖率提升
- 安全：漏洞检测率、误报率、平均修复时间

### 常见故障与恢复

| 故障模式                | 恢复策略                                           |
| ----------------------- | -------------------------------------------------- |
| 生成代码无法编译        | 迭代修复循环：编译→解析错误→修复→重试（最多 N 次） |
| Agent 建议已弃用 API    | 锁定代码库依赖版本，验证实际安装包 API             |
| 引入安全漏洞            | 预提交安全扫描 hook、敏感文件强制安全审查          |
| 测试生成创建不稳定测试  | 确定性测试模式、显式 Mock、重试检测和标记          |
| Agent 修改错误文件/范围 | 范围受限的上下文窗口、多文件变更确认提示           |

## 7. 用户运营 (User Operations)

### 核心 Agent 工作流

- **分群 Agent**：分析用户行为数据（事件/交易/互动），使用 RFM 分析、行为聚类和预测属性创建动态分群，维护分群定义和成员
- **生命周期管理 Agent**：将用户映射到生命周期阶段（获取→激活→留存→变现→推荐），在每个阶段触发适当干预，个性化触点
- **流失预测 Agent**：从行为信号（参与度下降/支持工单/功能放弃）构建和维护流失模型，生成高风险用户列表和干预建议
- **营销自动化 Agent**：设计和执行多触点互动活动（Push/邮件/应用内/短信），管理发送时间优化、个性化和频次上限
- **群组分析 Agent**：按获客渠道/时间/行为定义群组，追踪留存曲线，识别高价值群组模式，为产品和营销团队生成洞察

### 关键工具/集成

| 类别       | 具体工具                                                              |
| ---------- | --------------------------------------------------------------------- |
| CDP/分析   | Segment, Amplitude, Mixpanel, 神策数据, GrowingIO, 友盟               |
| 营销自动化 | Braze, CleverTap, 极光, 个推, Iterable, Customer.io                   |
| 通信       | Twilio（短信）, SendGrid（邮件）, APNs/FCM（Push）, 微信/企业微信 API |
| A/B 测试   | Optimizely, LaunchDarkly, Firebase Remote Config, VWO                 |
| 数据仓库   | 用户级分析——Snowflake, BigQuery, ClickHouse                           |

### 领域风险画像

- **过度沟通**：过多 Push/邮件导致用户厌烦和退订
- **隐私侵犯**：在未经同意的情况下基于敏感行为定向（如健康 App 使用模式）
- **个性化失败**：错误的个性化（向经历丧失的人发送婴儿用品推广）
- **歧视**：分群无意中歧视受保护群体
- **指标作弊**：Agent 为代理指标优化而不与商业价值对齐（如通过烦人通知虚增 DAU）

### 数据敏感度分级

- **PII（高）**：用户画像、联系方式、与可识别用户关联的行为数据
- **敏感行为**：位置数据、健康/健身数据、金融行为、浏览历史
- **聚合（低）**：群组级指标、匿名漏斗数据

### 监管/合规要求

- **PIPL（中国）**：数据收集显式同意、目的限制、数据最小化、跨境传输限制、删除权
- **GDPR**：营销通信同意、反对 Profiling 权、高风险处理的 DPIA
- **CAN-SPAM/TCPA（美国）**：退订合规、短信同意要求
- **平台政策**：iOS App Tracking Transparency、Google 同意要求

### 性能/延迟要求

- **分群更新**：触发式活动准实时（<5分钟延迟），批量分群每日
- **活动触发**：实时事件驱动触发从事件到消息投递 <1分钟
- **流失预测**：每日模型评分足够；高价值用户实时
- **A/B 测试结果**：统计显著性监控、每日报告

### HITL 要求

- **强制**：活动内容审批、敏感属性新分群创建、通知频率策略变更、激励活动预算分配
- **建议**：流失干预策略审核、A/B 测试启动和结论决策、新渠道接入

### 评估指标

- 留存率（D1/D7/D30）、流失率、唤醒率
- DAU/MAU 比率（粘性）、会话频率、功能采纳率
- 活动：打开率、CTR、转化率、退订率、退出率
- LTV（生命周期价值）、LTV/CAC 比率
- NPS、CSAT
- A/B 测试速度和胜率

### 常见故障与恢复

| 故障模式                | 恢复策略                                              |
| ----------------------- | ----------------------------------------------------- |
| 通知疲劳（退订激增）    | 频次上限、参与度评分门控、自动冷静期                  |
| 错误个性化（尴尬/冒犯） | 内容安全审查、回退到通用消息、敏感话题检测            |
| 流失模型误报            | 分级干预（先低成本）、A/B 测试干预措施、反馈至模型    |
| 活动发送到错误分群      | 发送前分群大小验证、沙箱测试、渐进发布（1%→10%→100%） |
| 未尊重同意/退出         | 实时偏好中心同步、发送层硬性退出强制执行              |

## 8. 行业调研 (Industry Research)

### 核心 Agent 工作流

- **市场分析 Agent**：从多来源（金融数据库/新闻/政府统计/行业报告）收集数据，综合市场规模、增长率、竞争格局和价值链分析为结构化报告
- **竞争情报 Agent**：监控竞争对手活动（产品发布/定价变动/招聘动态/专利申请/监管提交），维护竞品档案，重大变化时生成告警
- **趋势预测 Agent**：从专利/学术论文/创业融资/社交媒体/政策文件等信号中识别新兴趋势，附带置信度的前瞻性分析
- **报告生成 Agent**：将研究发现转化为结构化报告（执行摘要/方法论/发现/建议），多格式输出，保持引用严谨和来源归属
- **监管追踪 Agent**：跨辖区追踪监管变化，评估业务影响，生成合规差距分析，维护监管变更日历

### 关键工具/集成

| 类别      | 具体工具                                                              |
| --------- | --------------------------------------------------------------------- |
| 数据源    | Wind（万得）, Bloomberg, Statista, IBISWorld, 国家统计局, Euromonitor |
| 新闻/媒体 | NewsAPI, GDELT, 财新/36kr API, 社交聆听工具                           |
| 专利      | WIPO, USPTO, CNIPA（国家知识产权局）, Google Patents                  |
| 财务报告  | SEC EDGAR, 巨潮资讯网, Hong Kong ExFact                               |
| 网页采集  | Scrapy, Playwright（动态内容）, SerpAPI（搜索）                       |
| NLP       | 情感分析模型、NER（公司/人物提取）、摘要生成                          |

### 领域风险画像

- **来源可靠性**：将不可靠来源与权威来源混合，或将 AI 生成分析呈现为事实研究
- **幻觉**：捏造统计数据、市场规模或引述——对研究信誉是灾难性的
- **时效性**：使用过期数据做当前市场声明
- **版权**：过度复制有版权的报告内容
- **偏见**：趋势分析中的确认偏误、竞争分析中的幸存者偏误

### 数据敏感度分级

- **机密**：专有研究发现、客户特定分析、竞争策略建议
- **授权许可**：第三方数据（Bloomberg/Wind）——许可证限制再分发
- **公开**：政府统计、已公开报告、公开新闻

### 监管/合规要求

- **证券**：可能被视为投资建议的研究可能触发证券法规（中国《证券法》、美国 Investment Advisers Act）
- **数据许可**：遵守 Bloomberg/Reuters/Wind 数据再分发条款
- **版权**：摘要有版权报告时的合理使用考量
- **反竞争**：竞争情报收集不得涉及非法手段（黑客/行贿/虚假陈述）

### 性能/延迟要求

- **告警生成**：突发新闻/监管变化准实时（<15分钟）
- **报告生成**：小时到天级（异步工作流），综合市场报告可能需要 Agent 数小时研究和综合
- **数据刷新**：市场数据每日、竞争监控每周、深度分析每季度

### HITL 要求

- **强制**：所有发布的研究必须经人工分析师审核；事实声明必须核实；定量声明必须引用来源；前瞻性声明必须适当加免责声明
- **建议**：来源选择审核、方法论验证、最终报告编辑

### 评估指标

- 完整性：相关来源覆盖率、竞争格局完整性
- 时效性：洞察产出时间、突发事件告警延迟
- 有用性：分析师节省时间、决策者满意度
- 原创性：超越表面聚合的新颖洞察

### 常见故障与恢复

| 故障模式             | 恢复策略                                             |
| -------------------- | ---------------------------------------------------- |
| 幻觉统计数据         | 所有定量声明强制来源引用、验证 Agent 交叉检查        |
| 陈旧数据当现数据呈现 | 所有数据点加时间戳、新鲜度检查、超阈值自动标记       |
| 版权侵犯（过度引用） | 归因摘要、合理使用指南强制、改写比率监控             |
| 遗漏关键竞品/发展    | 多来源交叉引用、缺口检测清单、人工范围审核           |

## 9. 学术调研 (Academic Research)

### 核心 Agent 工作流

- **文献综述 Agent**：搜索学术数据库（Semantic Scholar、PubMed、CNKI/知网、arXiv），按相关性检索和排序论文，提取关键发现，识别研究空白，生成带规范引用管理的结构化文献综述。
- **假设生成 Agent**：分析跨论文的现有发现，识别矛盾或未探索的交叉领域，提出可检验假设并建议实验方法。
- **实验设计 Agent**：给定假设后，提出实验设计（样本量计算、对照组、统计检验），识别潜在混淆变量，生成预注册文档。
- **数据分析 Agent**：处理研究数据集，执行统计分析（回归、ANOVA、生存分析），生成可视化图表，检查常见统计错误（p-hacking、多重比较）。
- **写作与发表 Agent**：辅助论文撰写（遵循期刊排版要求），生成摘要，管理参考文献（BibTeX/EndNote），准备投稿包。

| 预注册     | OSF（Open Science Framework）, AsPredicted                                            |

### 领域风险画像

- **引用捏造**：Agent 生成不存在的引用或错误归属——构成学术欺诈
- **统计错误**：错误分析导致虚假结论（错误检验选择、假设违反）
- **抄袭**：生成文本与源材料过于相似
- **p-Hacking/HARKing**：Agent 无意中帮助研究者挑选结果
- **可复现性危机**：Agent 辅助的研究无法被复现

### 数据敏感度分级

- **高**：人体受试者数据（IRB 监管）、患者数据（HIPAA）、未发表研究成果、基金申请书
- **中**：预发表稿件、初步结果、同行评审意见
- **低**：已发表论文、公开数据集、开放获取研究

### 监管/合规要求

- **研究伦理**：人体受试者研究需 IRB/伦理委员会批准、知情同意、数据匿名化
- **发表伦理**：COPE（出版伦理委员会）指南；Nature、Science 等期刊现已要求披露 AI 辅助使用
- **数据法规**：HIPAA（健康数据）、FERPA（教育数据）、多机构合作的数据共享协议
- **资助要求**：基金机构（NSF、NSFC/国自然、NIH）对数据管理计划和开放获取的要求

### 性能/延迟要求

- **文献搜索**：初始结果 <30s，全面检索分钟级
- **统计分析**：秒到分钟级（取决于数据集大小）
- **写作辅助**：实时或准实时（类似代码补全体验）
- **完整文献综述**：小时到天级（异步大规模检索和综合）

### HITL 要求

- **强制**：所有发表内容必须经人工研究者审核和批准；假设选择、实验设计审批、统计方法选择、结果解释、稿件投递
- **关键原则**：研究者必须承担知识产权——Agent 辅助但人类对科学声明负责

### 评估指标

- 文献覆盖率：相关论文召回率、搜索结果精确率
- 引用准确性：100% 可验证引用（零捏造容忍）
- 统计正确性：检验选择适当性、假设检查
- 写作质量：可读性、期刊风格遵从度、清晰度评分
- 可复现性：可从提供材料复现的分析比例
- 研究者生产力：文献综述/分析/写作节省时间

### 常见故障与恢复

| 故障模式           | 恢复策略                                                |
| ------------------ | ------------------------------------------------------- |
| 引用捏造           | 每条引用必须解析到真实论文——自动 DOI/数据库验证         |
| 统计方法误用       | 假设检查 Agent 层、分析前诊断测试、人工统计师审核       |
| 生成文本抄袭       | 集成查重工具（Turnitin/iThenticate）、原创性评分        |
| 遗漏相关文献       | 多数据库搜索、引用链追踪（前向/后向）、人工专家覆盖审核 |
| 未发表成果数据泄露 | 严格访问控制、不在机密研究数据上训练、隔离处理环境      |

## 10. 企业知识库 (Enterprise Knowledge Base)

### 核心 Agent 工作流

- **文档处理 Agent**：摄入多格式文档（PDF、Word、PPT、Confluence 页面、邮件、会议纪要），提取结构化知识，适当分块以支持检索，维护文档元数据。
- **知识图谱 Agent**：从非结构化文本中提取实体和关系，构建和维护知识图谱（人物、项目、技术、流程），解决实体歧义，关联跨域知识。

- **知识缺口分析 Agent**：识别未文档化的流程、陈旧内容、文档间矛盾、以及用户查询持续无法找到好答案的领域。

### 性能/延迟要求

- **文档摄入**：分钟到小时级（取决于量）；批处理可接受
- **知识图谱更新**：关键文档准实时，通用更新每日批量
- **可用性**：工作时间高可用（99.9%）

### HITL 要求

- **强制**：访问控制策略定义、敏感文档分级、知识库纠错（Agent 回答错误时）、内容退役决策
- **建议**：新文档源接入、面向外部的 FAQ 审批、关键领域知识图谱关系验证

### 评估指标

- 搜索相关性：MRR（平均倒数排名）、NDCG、precision@k
- 答案质量：忠实度（有源文档支撑）、完整性、引用准确性
- 覆盖率：组织知识捕获比例、查询解决率
- 新鲜度：平均文档年龄、陈旧内容检测率
- 使用量：查询量、用户满意度（点赞/点踩）、重复查询率（越低越好）
- 访问控制：零未授权访问事件

| ------------------------ | -------------------------------------------------------- |

## 11. 财务 (Finance & Accounting)

### 核心 Agent 工作流

- **发票处理 Agent**：接收发票（邮件、扫描件、电子发票），执行 OCR 提取，与采购订单和验收单三单匹配，路由审批，过账到 ERP。
- **费控 Agent**：审核报销单的政策合规性（差旅标准、票据要求、品类规则），标记异常，合规内自动审批，异常路由人工审核。
- **财务报告 Agent**：从子账簿汇总数据，执行合并（多实体、多币种），生成试算平衡表、利润表、资产负债表、现金流量表；运行与预算和上期的差异分析。
- **税务合规 Agent**：计算税务负债（增值税、企业所得税、代扣代缴税），生成纳税申报，管理转让定价文档，跨辖区追踪法规变更。
- **对账 Agent**：跨系统匹配交易（银行对账单 vs 总账、公司间、子账簿到总账），识别差异，建议解决方案，生成对账报告。
- **预算与预测 Agent**：基于历史数据+业务驱动因素构建预测，执行情景分析，跟踪预算 vs 实际差异，生成滚动预测。

### 关键工具/集成

| 类别     | 具体工具                                                  |
| -------- | --------------------------------------------------------- |
| ERP      | SAP S/4HANA, Oracle ERP Cloud, 用友 U8/NC, 金蝶 K/3/Cloud |
| 费控     | SAP Concur, Expensify, Ramp, 内部报销系统                 |
| 税务     | Thomson Reuters ONESOURCE, Avalara, 航天信息（金税系统）  |
| 银行     | 银行 API（PSD2/Open Banking）, SWIFT, 银企直连            |
| OCR/文档 | ABBYY, Kofax, 百度AI/腾讯AI OCR（中文发票）               |
| BI       | Tableau, Power BI, 帆软 FineReport                        |

### 领域风险画像

- **财务错报**：财务报告错误导致监管处罚、投资者诉讼、审计保留意见
- **欺诈**：虚假发票、幽灵员工、报销单操纵
- **税务错误**：税款计算错误导致罚款和滞纳金，或多缴税
- **公司间差异**：多实体组织合并错误
- **汇率/截止错误**：错误汇率或会计期间截止错误

### 数据敏感度分级

- **极度机密**：未公开财务结果、高管薪酬、并购估值、税务头寸
- **机密**：总账数据、供应商合同、员工报销数据、银行账户信息
- **受监管**：所有财务记录受 SOX（上市美股公司）和审计留存要求约束（通常7-10年）

### 监管/合规要求

- **中国**：企业会计准则（CAS）、增值税暂行条例、金税四期电子税务监控、年度审计要求
- **美国**：US GAAP、SOX 第404条（内部控制）、IRS 法规、ASC 606（收入确认）
- **国际**：跨国报告的 IFRS、转让定价文档（OECD BEPS）、国别报告
- **审计**：要求完整审计轨迹——每笔交易必须可追溯到原始凭证

### 性能/延迟要求

- **发票处理**：OCR 和匹配 <1分钟，当日过账
- **月结**：目标从 10+ 天缩短至 3-5 天，批处理必须在月结窗口内完成
- **税务申报**：必须满足严格的监管截止日期（如中国增值税每月15日前申报）
- **对账**：银行对账每日，其他对账每月
- **报告**：按需即席查询 <30s，定时报表在批处理窗口内

### HITL 要求

- **强制**：超阈值记账凭证、财务报表签字（CFO/财务总监）、纳税申报提交、审计应答、坏账核销、公司间定价决策、自动匹配规则的任何覆盖
- **监管**：SOX 要求关键控制的文档化审核和审批；必须执行职责分离（创建人≠审批人）

### 评估指标

- 处理效率：每小时处理发票数、直通处理率（零人工介入）
- 准确性：总账过账准确率、对账匹配率、税款计算准确率
- 时效性：月结天数、报告交付及时性、申报截止日期遵从
- 合规性：审计发现、控制例外、职责分离违规
- 成本节约：人员重新分配、错误相关罚款减少

### 常见故障与恢复

| 故障模式         | 恢复策略                                                  |
| ---------------- | --------------------------------------------------------- |
| OCR 误读发票金额 | 置信度评分+低于阈值人工审核、三单匹配作为验证             |
| 会计期间归属错误 | 期末截止规则、与采购订单/交货日期验证、反向记账凭证能力   |
| 合并汇率错误     | 汇率源验证（央行/ECB）、折算与重新计量对账                |
| 税款计算错误     | 多方法验证、与上期比较、税务机关税率表验证                |
| 重复付款         | 重复检测（供应商+金额+日期+发票号）、潜在重复的审批工作流 |

## 12. 法务 (Legal)

### 核心 Agent 工作流

- **合同审查 Agent**：逐条分析合同对照标准条款手册，识别偏离已批准条款之处，标记风险条款（无限责任、不利赔偿、自动续约、竞业禁止），建议谈判立场，生成红线标注。
- **监管合规 Agent**：跨辖区监控法规变化，将法规映射到业务流程，执行差距分析，生成合规检查清单，跟踪整改状态。
- **诉讼支持 Agent**：执行电子发现文档审查（相关性分类、特权检测），组织案件事实，生成案件时间线，识别关键文档，辅助法律研究。
- **知识产权管理 Agent**：监控商标/专利申请，追踪续期截止日，执行自由实施检索（FTO），识别潜在侵权，管理 IP 组合。
- **尽职调查 Agent**：在并购中审查目标公司文件，从重大合同中提取关键条款，识别负债和或有事项，生成尽调报告，标记危险信号。
- **政策起草 Agent**：根据辖区要求生成隐私政策、服务条款、员工手册、合规政策，维护版本历史。

### 关键工具/集成

| 类别     | 具体工具                                                |
| -------- | ------------------------------------------------------- |
| 合同管理 | DocuSign CLM, Ironclad, 法大大, 上上签, Icertis         |
| 法律研究 | Westlaw, LexisNexis, 北大法宝, 威科先行, 中国裁判文书网 |
| 电子发现 | Relativity, Nuix, Logikcull, DISCO                      |
| 知识产权 | Thomson Reuters IP, MaxVal, 国家知识产权局系统          |
| 合规     | OneTrust（隐私）, LogicGate（GRC）, SAI360              |
| 文档比对 | Litera Compare, BlackBoiler, Contract AI 工具           |

### 领域风险画像

- **遗漏风险条款**：Agent 未能识别不利条款，导致重大财务敞口
- **错误法律意见**：Agent 提供错误法律解释——公司/律所责任
- **特权放弃**：电子发现过程中无意披露受保护通信
- **辖区错误**：在多辖区事务中适用了错误辖区的法律
- **截止日期管理失败**：错过申报期限（诉讼时效、IP 续期、监管提交）
- **保密性违反**：法律文件是任何组织中最敏感的文档之一

### 数据敏感度分级

- **律师-客户特权**：法律意见、诉讼策略、和解讨论——最高保护级别
- **极度机密**：并购文件、监管调查档案、IP 商业秘密、劳动争议
- **机密**：标准合同、政策、合规记录
- **受监管**：法院文书（部分公开）、监管提交、隐私相关记录

### 监管/合规要求

- **职业伦理**：非律师不得提供"法律意见"——Agent 只能提供供律师审核的法律"信息"或"分析"
- **数据隐私**：GDPR（欧盟）、PIPL（中国）、CCPA（美国）——法务团队须确保公司合规并确保自身数据处理合规
- **电子发现**：美国联邦民事诉讼规则、诉讼保留义务、比例原则要求
- **中国专项**：民法典、反垄断法、数据安全法、个人信息保护法
- **跨境**：国际条约、海牙公约（文书送达）、司法互助条约（刑事事务）

### 性能/延迟要求

- **合同审查**：标准合同 <1小时（异步），复杂多方协议 <24小时
- **监管监控**：每日扫描监管来源，关键变化实时告警
- **电子发现**：须处理数百万文档——吞吐量比延迟更重要，每小时数千文档
- **法律研究**：初始判例检索 <30s，全面研究备忘录分钟级
- **IP 监控**：每周侵权扫描，关键商标每日

### HITL 要求

- **强制**：**所有法律输出在采取行动前必须经执业律师审核**。合同谈判、诉讼策略、监管申报提交、法律意见、特权判定。
- **本域在12个域中 HITL 要求最高**——因为非授权执业本身即违法，且法律错误具有放大效应。

### 评估指标

- 合同审查：问题检测召回率（必须捕获所有风险条款）、误报率、审查时间缩减
- 合规：法规覆盖率、缺口检测准确率、整改追踪完整性
- 电子发现：召回率（必须找到所有相关/特权文档）、精确率、审查速度（文档/小时）
- 法律研究：引用判例相关性、法律分析准确性、全面性
- 通用：律师满意度、时间节约、错误率（生产中遗漏的问题）、截止日期遵从

| ---------------- | ---------------------------------------------------------------- |

## 13. 在线直播 (Live Streaming)

### 核心 Agent 工作流

- **直播编排 Agent**：管理直播全生命周期，包括推流初始化、多平台分发（抖音/快手/B站/视频号）、转码配置、CDN 调度、封面与标题优化。流程：场景配置 → 推流启动 → 多平台同步分发 → 实时质量监控 → 异常自动切换 → 回放生成。
- **互动运营 Agent**：实时分析弹幕/评论情感，管理互动玩法（红包雨、答题、投票、连麦排队），根据热度曲线自动调整互动节奏，维护粉丝等级和特权体系。流程：弹幕摄入 → 情感分析 → 互动策略匹配 → 玩法触发 → 效果反馈。
- **实时内容审核 Agent**：对视频流、音频流和弹幕进行实时合规审核，检测违规内容（色情/暴力/政治敏感/广告引流/未成年人保护），执行自动处置（警告/禁言/断流）。流程：多模态流采样 → AI 分类 → 风险评分 → 自动处置/人工复审。
- **直播电商转化 Agent**：管理直播带货全链路，包括商品上下架节奏、优惠券发放时机、库存锁定、实时销售看板，根据观众行为动态调整讲解顺序和促销力度。流程：选品排期 → 实时上架 → 促销触发 → 库存同步 → 订单归因。
- **数据分析 Agent**：实时追踪关键指标（在线人数/互动率/转化率/礼物收入/GPM），生成直播复盘报告，对比历史场次，输出优化建议。

### 领域风险画像

- **违规内容传播**：直播实时性极强，违规内容一旦播出即造成不可逆影响，审核延迟直接导致监管处罚
- **未成年人保护**：未成年主播或观众参与打赏/不良内容互动，触发严格监管
- **虚假宣传**：直播带货中夸大商品功效、虚标价格、刷单造假
- **资金安全**：礼物/打赏系统存在洗钱、诈骗、诱导消费风险
- **技术故障**：推流中断、CDN 故障、延迟飙升导致用户流失和商业损失
- **数据泄露**：主播/观众隐私数据、交易数据泄露

### 数据敏感度分级

- **高（PII+金融）**：用户实名认证信息、支付账户、打赏/交易记录、主播收入数据
- **机密**：运营策略、选品数据、MCN 合同、平台分成比例、算法推荐参数
- **内部**：聚合观看数据、互动统计、公开直播内容回放
- **实时流数据**：视频/音频流本身包含主播肖像和环境信息，需按场景分级保护

### 监管/合规要求

- **中国**：《互联网直播服务管理规定》、《网络直播营销管理办法（试行）》、《未成年人保护法》直播相关条款、网信办"清朗"行动、《广告法》适用于直播带货

- **电商合规**：直播带货需明示商品信息、七天无理由退货适用、禁止虚构原价
- **跨境**：COPPA（美国儿童保护）、DSA（欧盟数字服务法案）、各平台社区准则
- **税务**：主播收入个人所得税代扣、MCN 机构税务合规

### 性能/延迟要求

- **推流延迟**：低延迟直播 <1s，超低延迟（连麦）<400ms，标准直播 <3s
- **内容审核**：视频帧审核 <500ms，弹幕审核 <200ms（同步过滤）
- **互动响应**：红包/投票/连麦操作 <1s 响应
- **电商操作**：商品上下架 <2s，库存锁定 <500ms
- **可用性**：直播期间 99.99%，推流中断直接导致商业损失

### HITL 要求

- **强制**：高风险内容审核复审（涉政/涉恐）、大额打赏人工确认、直播带货违规处置（断流/封禁）、未成年人保护相关处置、主播资质审核
- **建议**：选品审核、促销方案审批、新互动玩法上线前测试、直播复盘中的策略调整建议
- **实时监控**：运营人员须有直播间实时监看能力和一键断流权限

### 评估指标

- 观看指标：PCU（峰值在线）、ACU（平均在线）、观看时长、留存曲线
- 互动指标：弹幕密度、互动率、礼物收入、付费用户占比
- 电商指标：GPM（千次观看成交额）、转化率、客单价、退货率
- 审核指标：违规内容检出率、误报率、平均处置延迟、漏检率
- 技术指标：卡顿率、首帧时间、推流成功率、CDN 命中率
- 商业指标：单场 GMV、ROI（投流回报）、粉丝增长率

## 14. 广告素材制作 (Creative/Content Production)

### 核心 Agent 工作流

- **创意生成 Agent**：基于营销 Brief（目标受众、卖点、调性）自动生成多模态广告素材，包括文案、图片、短视频脚本和落地页。流程：Brief 解析 → 创意策略生成 → 多格式素材生产 → 品牌一致性校验 → A/B 变体输出。
- **品牌合规检查 Agent**：审核生成素材是否符合品牌规范（Logo 使用规则、色彩体系、字体规范、调性词库、禁用元素），检查广告法合规性（绝对化用语、虚假宣传），输出合规评分和修改建议。流程：素材摄入 → 品牌规范匹配 → 法规校验 → 风险标注 → 修改建议。
- **素材适配 Agent**：将主创意自动适配为多平台多尺寸版本（抖音竖版9:16、朋友圈1:1、Banner横版16:9、信息流3:4），智能裁切、文字重排、元素重组，保持视觉完整性。流程：源素材解析 → 平台规格匹配 → 自动适配 → 质量检查 → 批量输出。
- **效果预测 Agent**：基于历史投放数据和素材特征（色调、文案情感、CTA 类型、人物/产品占比）预测素材 CTR/CVR，排序素材优先级，建议优化方向。
- **工作流管理 Agent**：协调创意制作全流程，管理需求池、任务分配、审批流转、版本管理、素材资产库，追踪从 Brief 到上线的全链路状态。

## 15. 游戏开发 (Game Development)

### 数据敏感度分级

- **极度机密**：未公布游戏设计文档、核心玩法专利、源代码、未发布美术资产

## 18. 供应链与物流 (Supply Chain & Logistics)

### 核心 Agent 工作流

- **需求预测 Agent**：整合历史销售数据、市场趋势、促销计划、天气/节假日因素，运行预测模型（ARIMA、Prophet、DeepAR、Transformer），生成多层级（SKU/仓库/区域）需求预测，输出安全库存建议。流程：数据汇聚 → 特征工程 → 模型集成 → 预测生成 → 异常校正 → 库存建议。
- **库存优化 Agent**：基于需求预测和供应约束，计算最优库存策略（再订货点/经济订货量/安全库存），管理多仓库间库存调拨，优化库存周转和服务水平的平衡。
- **路径规划 Agent**：优化运输路径和配送计划，考虑车辆容量、时间窗口、交通状况、成本目标，支持实时动态调整（新订单插入/异常改道）。流程：订单池汇聚 → 约束建模 → 路径优化（VRP 求解）→ 调度派发 → 实时监控与改道。
- **供应商评估 Agent**：对供应商进行多维度评估（质量/交付/价格/响应速度/ESG），分析供应商风险（财务健康/地缘政治/单一来源），辅助采购决策和供应商多元化。
- **关务合规 Agent**：管理进出口合规，包括 HS 编码分类、关税计算、原产地规则验证、制裁筛查、报关单生成、贸易协定利用率优化。

- **预测偏差**：需求预测严重偏离导致库存积压（资金占用）或缺货（销售损失）
- **供应链中断**：自然灾害、地缘政治、疫情等导致供应链断裂
- **路径优化失败**：异常路径导致运输延误、成本超支或货物损毁
- **关务违规**：HS 编码分类错误导致关税补缴、罚款甚至货物扣押
- **供应商风险**：关键供应商单一来源，财务恶化或突发事件导致断供
- **库存数据不准**：系统库存与实物不一致导致连锁决策错误

### 数据敏感度分级

- **机密**：供应商合同条款和定价、采购成本、库存策略参数、需求预测模型
- **商业敏感**：库存水位、物流路径、仓库布局、供应商评估结果
- **受监管**：海关申报数据、原产地证明、危险品运输记录（需保留5-10年）
- **IoT 数据**：GPS 轨迹、温湿度记录——可能涉及位置隐私

### 监管/合规要求

- **海关法规**：中国《海关法》《进出口关税条例》、WCO HS 分类规则、各国海关申报要求
- **贸易合规**：出口管制（EAR/美国商务部 BIS、中国《出口管制法》）、制裁筛查（OFAC SDN、欧盟制裁名单）
- **运输法规**：危险品运输（IATA DGR 航空、IMDG 海运、ADR 陆运）、冷链运输（FDA 21 CFR Part 11、GSP）
- **数据与隐私**：PIPL/GDPR 适用于供应链中的个人数据、跨境数据传输限制
- **ESG/可持续性**：碳排放报告（CBAM 欧盟碳边境调节机制）、供应链尽职调查（欧盟 CSDDD）

### 性能/延迟要求

- **需求预测**：每日批量更新（隔夜跑批），促销/突发事件触发即时重算 <30分钟
- **库存优化**：补货建议每日生成，紧急补货 <1小时
- **路径规划**：初始规划 <5分钟（数百订单），实时改道 <30s
- **关务申报**：单票报关 <10分钟（自动分类+制裁筛查），批量申报小时级
- **IoT 监控**：温湿度/GPS 异常告警 <1分钟

### HITL 要求

- **强制**：大额采购订单审批、新供应商准入审核、关务申报异常处理、危险品运输审批、库存报废决策、供应链中断应急方案选择
- **建议**：需求预测异常值确认、库存调拨方案审核、路径规划重大调整
- **紧急**：供应链中断事件需人工指挥应急响应，Agent 辅助方案生成和执行跟踪

### 评估指标

- 预测：预测准确率（MAPE/WMAPE）、偏差率、预测增值（vs 朴素方法）
- 库存：库存周转率、缺货率、库存持有成本、服务水平（Fill Rate）

- 关务：HS 分类准确率、关税优化节约金额、申报一次通过率

## 19. 医疗健康 (Healthcare)

### 核心 Agent 工作流

- **临床决策支持 Agent**：整合患者病历（主诉、既往史、检验结果、影像报告），运行临床推理引擎，匹配临床指南（UpToDate/中国临床路径），输出鉴别诊断列表与建议检查方案。流程：病历摄入 → 结构化提取 → 临床推理 → 指南匹配 → 鉴别诊断排序 → 医师审核。
- **智能分诊 Agent**：接收患者自述症状，通过标准化问诊流程收集症状、严重度和持续时间，使用 ESI/Manchester Triage 评估分诊级别，路由至对应科室。
- **病历分析 Agent**：对非结构化病历执行 NLP 提取（诊断、用药、手术史、过敏信息），构建患者时间线，识别信息缺失和矛盾，生成结构化摘要。
- **药物交互检查 Agent**：接收当前用药列表与拟新增处方，查询药物相互作用数据库（DrugBank/MCDEX），评估严重程度，标记肝肾功能禁忌，计算剂量合理性。
- **医学影像分析 Agent**：接收 DICOM 影像（X光/CT/MRI），运行预训练影像模型（肺结节检测、骨折识别、眼底病变），标注可疑区域，生成结构化报告草稿。

### 领域风险画像

- **误诊/漏诊**：Agent 输出错误鉴别诊断，医师过度依赖导致延误治疗——直接威胁患者生命安全
- **药物不良事件**：交互检查遗漏导致严重药物反应
- **影像漏检**：AI 模型遗漏恶性病灶（假阴性），延误癌症等重大疾病诊断
- **隐私泄露**：患者健康信息（PHI）泄露——医疗数据泄露平均成本全行业最高
- **算法偏见**：训练数据中人群代表性不足导致对特定群体诊断准确率下降
- **过度依赖**：临床人员丧失独立临床判断能力，形成自动化偏差

### 数据敏感度分级

- **极度敏感（PHI）**：患者身份信息、诊断记录、基因组数据、精神科/HIV/生殖记录（特殊保护类别）
- **机密**：临床决策模型参数、医院运营数据、科研中间数据、医师绩效数据
- **内部**：脱敏后的聚合统计数据、公开临床指南、药品说明书信息

### 监管/合规要求

- **中国**：《医疗器械监督管理条例》（AI 辅助诊断软件按 II/III 类医疗器械监管，需 NMPA 注册）、《基本医疗卫生与健康促进法》、三级等保要求、《人类遗传资源管理条例》
- **美国**：HIPAA（隐私/安全/违规通知规则）、FDA 510(k)/De Novo（AI/ML 医疗器械审批，SaMD 框架）、HITECH Act
- **欧盟**：MDR（医疗器械法规 2017/745）、EU AI Act（医疗 AI 归为高风险）、GDPR 健康数据特殊条款
- **通用**：临床验证要求——AI 辅助诊断必须有多中心临床试验数据；上市后监测义务
- **数据出境**：中国《数据安全法》和《个人信息保护法》对医疗数据跨境传输有严格限制

### 性能/延迟要求

- **急诊分诊**：危险信号识别 <5s（红区患者零延迟容忍），完整分诊 <30s
- **药物交互检查**：处方校验 <2s（嵌入医嘱系统同步流程）
- **影像分析**：单张 X光 <30s，CT 序列 <5min，需支持 GPU 加速推理
- **临床决策支持**：建议生成 <10s（门诊场景医师等待时间有限）
- **系统可用性**：99.99%（医疗系统不可中断，急诊 7×24 运行）

### HITL 要求

- **强制**：所有诊断建议必须经执业医师确认后方可记录；处方必须经医师签发；影像 AI 标注必须经影像科医师审核出具正式报告；分诊结果作为建议而非最终决定
- **建议**：AI 模型性能漂移监控需定期临床专家审核；患者投诉和不良事件回溯分析
- **法律底线**：中国《执业医师法》和各国医师法明确规定——临床决策最终责任在执业医师，AI 仅为辅助工具

### 评估指标

- 诊断准确性：敏感度（Sensitivity）、特异度（Specificity）、AUC-ROC、阳性/阴性预测值
- 影像分析：病灶检测召回率、假阳性率、与金标准（病理）一致性
- 药物检查：交互检测覆盖率、临床显著交互召回率、告警疲劳度（过度告警率）
- 分诊准确性：与专家分诊一致率、过分诊/欠分诊率、危重患者识别延迟
- 运营效率：门诊等待时间缩减、病历录入时间节省

### 常见故障与恢复

| 故障模式                           | 恢复策略                                                                         |
| ---------------------------------- | -------------------------------------------------------------------------------- |
| 影像 AI 系统性漏检特定病灶类型     | 多模型集成投票、定期用新标注数据回归测试、漏检反馈闭环至模型重训                 |
| 药物数据库更新延迟导致新药交互遗漏 | 多数据源交叉校验（DrugBank+MCDEX+FDA）、药品上市公告自动触发数据库刷新           |
| EMR 集成中断导致无法获取患者历史   | 本地缓存关键数据、降级为手动输入模式、自动重连和数据补同步                       |
| 分诊模型对罕见急症识别不足         | 危险信号硬编码规则兜底（胸痛+呼吸困难=立即红区）、低置信度强制人工复核           |
| 患者隐私数据意外暴露               | 实时 PHI 检测和脱敏、访问审计告警、72小时内监管通报（HIPAA Breach Notification） |

## 20. 教育培训 (Education & Training)

### 核心 Agent 工作流

- **学习路径优化 Agent**：评估学员当前知识水平（前测/历史学习数据），结合学习目标和知识图谱，生成个性化学习路径（知识点序列、难度梯度、资源推荐），动态调整进度。
- **内容生成 Agent**：根据教学大纲和知识点，自动生成教学材料（讲义、练习题、案例分析、课件），支持多难度级别和多语言，确保教学法原则（Bloom 认知层次）的正确应用。
- **智能评测 Agent**：自动生成评测题目（选择/填空/主观/编程），执行自动评分，生成个性化反馈和薄弱项分析，支持自适应测试（CAT）。
- **智能辅导 Agent**：与学员进行一对一对话式辅导，采用苏格拉底式提问引导思考而非直接给答案，识别学员困惑点，提供分步解释和类比。
- **学情分析 Agent**：汇总学习行为数据（学习时长、完成率、错题分布、参与度），构建学员画像，预测学习风险（辍学/挂科），生成教师干预建议。

| LMS 平台  | Moodle、Canvas、Blackboard、学堂在线、中国大学MOOC、Coursera API         |

| 评测引擎  | Gradescope、Turnitin（查重）、CodeGrader（编程评测）、OJ（在线评测系统） |

### 领域风险画像

- **教学质量退化**：AI 生成内容存在知识性错误，学员学到错误知识且不自知
- **评测不公**：AI 评分对特定表达方式/文化背景存在偏见，主观题评分不一致
- **过度依赖**：学员依赖 AI 辅导获取答案而非真正理解，削弱自主学习能力
- **隐私风险**：未成年学员数据的特殊保护需求
- **学术诚信**：学员使用 AI 辅导 Agent 代写作业、论文——冲击学术诚信体系

### 数据敏感度分级

- **高（未成年人数据）**：学生身份信息、学习行为记录、成绩数据、心理评估结果
- **机密**：试题库（未公开）、评分标准、教学算法模型参数
- **内部**：课程大纲、已公开教学材料、聚合学情统计

### 监管/合规要求

- **中国**：《未成年人保护法》（网络保护专章）、《个人信息保护法》对不满14周岁未成年人的单独同意要求、双减政策合规
- **美国**：FERPA（家庭教育权和隐私法案）、COPPA（13岁以下儿童在线隐私）、ADA/Section 508（无障碍）
- **欧盟**：GDPR 儿童数据特殊保护（第8条）、EU AI Act（教育 AI 归为高风险）
- **通用**：教育内容必须符合课程标准、无歧视和偏见内容、支持多语言和无障碍访问

### 性能/延迟要求

- **智能辅导**：对话响应 <3s（超时学生注意力流失）
- **自适应测试**：题目推荐 <1s（考试流畅性要求）
- **内容生成**：单知识点讲义 <1min，完整课程生成小时级（异步）
- **评测评分**：客观题即时反馈，主观题 AI 评分 <30s/篇

### HITL 要求

- **强制**：课程内容上线前必须经学科教师审核确认知识正确性；主观题 AI 评分需教师抽检复核；学习路径重大调整需教师确认
- **建议**：AI 辅导对话定期抽样审核质量；学情预警触发的干预方案需教师判断执行
- **特殊要求**：涉及未成年学员的内容推送和数据使用需获得家长/监护人同意

### 评估指标

- 学习效果：前后测成绩提升、知识点掌握率、学习目标达成率
- 辅导质量：对话有效性（学员理解度提升）、苏格拉底式引导比例
- 评测质量：评分一致性（AI vs 人工 Cohen's Kappa）、题目区分度、信度/效度
- 参与度：完课率、学习时长、互动频率、辍学率

| 主观题评分严重偏差          | 评分校准机制（标杆样卷）、低置信度自动转人工、多维度评分量表 |

| 辅导 Agent 直接给出作业答案 | 教学策略护栏（强制引导式回答）、作业场景检测、答案输出过滤   |
| 高并发考试场景系统过载      | 弹性扩容预案、题目本地缓存、降级为离线考试模式               |

## 21. 客户服务 (Customer Service)

### 核心 Agent 工作流

- **多渠道对话 Agent**：统一处理来自在线聊天、电话（ASR 转文本）、邮件、社交媒体的客户咨询，理解意图，检索知识库生成回答，维护多轮对话上下文。流程：渠道接入 → 意图识别 → 知识检索 → 回答生成 → 满意度确认 → 工单归档。
- **智能路由 Agent**：基于工单内容（意图分类、情感分析、紧急度）、客户属性（VIP 等级、历史工单、LTV）和坐席状态，执行最优路由分配，管理排队和溢出策略。
- **知识检索 Agent**：维护产品/服务知识库，从历史工单、产品文档、FAQ 和内部 Wiki 中检索最相关答案，支持语义搜索和精确匹配混合，自动识别知识库缺口。
- **质检评分 Agent**：对客服对话进行全量自动质检（合规用语、服务态度、问题解决度、流程遵从），生成评分卡，标记需人工复检的低分对话。
- **升级管理 Agent**：检测需要升级的场景（客户情绪激动、超出权限、技术问题、投诉/舆情风险），自动触发升级至主管/专家/跨部门，确保 SLA 不被违反。

| 类别       | 具体工具                                                                       |

| 全渠道平台 | Zendesk、Salesforce Service Cloud、Freshdesk、网易七鱼、智齿科技、Udesk        |
| 语音/CTI   | Genesys Cloud、Avaya、Amazon Connect、Twilio Voice、合力亿捷                   |

### 领域风险画像

- **回答幻觉**：Agent 编造不存在的政策、承诺无法兑现的补偿方案——构成合同承诺
- **情绪误判**：未识别客户愤怒/焦虑导致升级延迟，引发投诉升级和舆情事件
- **隐私泄露**：在对话中意外暴露其他客户信息或内部系统数据
- **一致性缺失**：不同渠道/不同时间对同一问题给出矛盾回答
- **过度自动化**：客户无法触达人工坐席，循环在 AI 对话中无法解决问题

### 数据敏感度分级

- **PII（高）**：客户姓名、电话、地址、账户信息、对话中涉及的支付/订单数据
- **机密**：内部定价策略、补偿权限矩阵、未公开产品计划、客户投诉详情

- **欧盟**：消费者权利指令、GDPR（数据访问/删除权）、数字服务法（DSA）

## 22. 内容审核与安全 (Content Moderation & Safety)

### 核心 Agent 工作流

- **多模态审核 Agent**：对用户生成内容（UGC）执行多模态审核——文本（违规词/语义理解/变体对抗）、图片（色情/暴力/政治敏感/OCR 文字提取）、视频（逐帧抽样+音频转写审核）、音频（ASR 转文本+声纹检测）。流程：内容接收 → 格式解析 → 多模型并行审核 → 规则引擎叠加 → 置信度分级 → 处置决策。
- **策略引擎 Agent**：管理和执行内容审核策略（平台自定义规则、法规要求、广告主安全需求），支持策略版本管理、灰度发布和 A/B 测试，自动将新法规要求转化为可执行审核规则。
- **申诉处理 Agent**：接收被处置用户的申诉请求，重新审查原始内容和处置决策，收集补充信息，生成复核建议（维持/撤销/改判），记录处理理由。
- **对抗检测 Agent**：识别恶意用户的规避手段（谐音/拼音/火星文/图片嵌入文字/语义伪装/隐喻暗示），持续学习新对抗模式，自动更新检测规则。
- **合规报告 Agent**：按监管要求生成审核报告（审核量、违规类型分布、处置时效、申诉统计），支持监管对接（网信办/Ofcom/FTC），管理证据留存。

| 类别          | 具体工具                                                                                |
| ------------- | --------------------------------------------------------------------------------------- |
| 文本审核      | 自研 NLP 模型（BERT/RoBERTa 微调）、阿里云内容安全、腾讯天御、Perspective API（Google） |
| 图片/视频     | 自研 CV 模型、PhotoDNA（CSAM 检测）、AWS Rekognition、百度内容审核、CLIP                |

### 领域风险画像

- **漏审（假阴性）**：违规内容通过审核上线——法律风险、用户伤害、监管处罚
- **误审（假阳性）**：合法内容被错误删除——用户体验损害、言论自由争议、创作者流失
- **对抗升级**：恶意用户持续进化规避手段，审核模型滞后于对抗手段
- **审核员心理健康**：人工审核员长期接触极端内容导致心理创伤（PTSD）
- **审核偏见**：模型对特定语言/方言/文化表达存在偏见
- **规模挑战**：日均数亿条 UGC，延迟与准确率的平衡是核心工程挑战

### 数据敏感度分级

- **极度敏感**：CSAM（儿童性虐待材料）——法律强制报告、专门处理流程、严格访问控制
- **高**：用户内容原始数据（包含 PII）、审核决策记录、举报人信息
- **机密**：审核策略规则（泄露后被恶意利用规避）、对抗检测模型参数
- **内部**：聚合审核统计、模型性能指标、公开的社区准则

### 监管/合规要求

- **中国**：《网络安全法》、《网络信息内容生态治理规定》、《互联网信息服务算法推荐管理规定》——平台主体责任，违规内容24小时内处置
- **美国**：Section 230（平台免责与例外）、FOSTA-SESTA（性交易内容）、NCMEC 强制报告（CSAM）
- **欧盟**：数字服务法（DSA）——透明度报告、算法审计、用户申诉权、研究者数据访问
- **通用**：CSAM 检测和报告是全球法律义务；恐怖主义内容（Christchurch Call）、虚假信息治理
- **行业自律**：GIFCT（全球互联网反恐论坛）哈希共享、Tech Coalition 反 CSAM

### 性能/延迟要求

- **发布前审核**：文本 <500ms，图片 <1s，短视频 <30s（用户发布体验不可有明显延迟）
- **吞吐量**：支撑日均数亿条内容审核，峰值弹性扩容
- **对抗响应**：新对抗模式从发现到规则上线 <4h（紧急）、<24h（常规）
- **申诉处理**：自动复核 <1h，含人工 <24h
- **CSAM 检测**：零延迟容忍——检测即阻断+立即报告

### HITL 要求

- **强制**：申诉复核最终裁决需人工审核员确认；CSAM 案件必须人工复核+法律合规团队介入；边界案例（讽刺/艺术/新闻报道豁免）必须人工判断；审核策略变更需安全团队审批
- **建议**：高置信度处置可全自动，中低置信度按梯度分流人工审核
- **审核员保护**：限制单人连续审核极端内容时间、提供心理健康支持、CSAM 内容仅限认证审核员处理

### 评估指标

- 准确率：精确率（误审率）、召回率（漏审率）、F1 值，按违规类型分类统计
- 时效性：从内容发布到审核完成的延迟、违规内容平均在线时长
- 对抗能力：对抗样本检测率、新对抗模式平均响应时间
- 用户体验：申诉成功率、申诉处理时效、创作者满意度
- 合规：监管通报数量、透明度报告指标、CSAM 报告时效

### 常见故障与恢复

| 故障模式                       | 恢复策略                                                               |
| ------------------------------ | ---------------------------------------------------------------------- |
| 新对抗手段大规模绕过审核       | 实时对抗样本收集、紧急规则热更新、临时提高审核严格度（宁误勿漏）       |
| 模型更新导致误审率飙升         | 灰度发布（1%→10%→100%）、自动回滚机制、A/B 测试验证                    |
| 审核系统宕机导致内容积压       | 降级策略（高风险内容排队、低风险临时放行）、多可用区容灾               |
| 合规时限将至但人工审核队列过长 | 动态优先级调整、临时扩充审核团队、AI 辅助预排序加速人工审核            |
| CSAM 检测系统漏检              | PhotoDNA + 多模型冗余检测、哈希库持续更新（NCMEC/GIFCT）、定期红队测试 |

## 23. IT 运维 SRE/DevOps (IT Operations)

### 核心 Agent 工作流

- **事件响应 Agent**：接收监控告警（Prometheus/PagerDuty/自定义），执行自动关联分析（告警聚合、拓扑关联、时序关联），推断根因假设，触发预定义自动修复（Runbook 自动化），升级至 On-Call 工程师。流程：告警接收 → 告警聚合 → 拓扑分析 → 根因假设 → 自动修复尝试 → 升级/闭环。
- **监控分析 Agent**：持续分析指标（CPU/内存/磁盘/网络/应用 APM）、日志（模式识别/异常检测）和链路追踪数据，建立动态基线，检测异常趋势，减少告警噪声。
- **部署自动化 Agent**：管理 CI/CD 流水线和发布流程，执行金丝雀发布（渐进式流量切换 + 指标监控），自动回滚失败部署，管理功能开关，协调跨服务依赖部署顺序。
- **容量规划 Agent**：基于历史负载模式和业务增长预测，建模资源需求（计算/存储/网络），生成扩容建议和预算预测，识别资源浪费，优化成本。
- **安全运营 Agent**：持续监控安全事件（IDS/IPS 告警、WAF 日志、漏洞扫描结果），执行威胁情报匹配，自动响应常见攻击（IP 封禁/账号锁定），管理漏洞修复优先级。

| 类别          | 具体工具                                                                            |
| ------------- | ----------------------------------------------------------------------------------- |
| 监控/可观测性 | Prometheus、Grafana、Datadog、New Relic、Splunk、阿里云 ARMS、Zabbix                |

### 领域风险画像

- **自动修复失误**：Agent 执行错误的自动修复操作（如误删 Pod、错误回滚、资源误释放），导致二次故障
- **告警风暴**：级联故障产生海量告警淹没团队，真正根因被噪声掩盖
- **部署回滚失败**：金丝雀检测不到的慢性问题（数据损坏/内存泄漏），回滚时数据不兼容
- **安全事件响应延迟**：Agent 误判安全告警优先级，延误对真实攻击的响应
- **成本失控**：自动扩容策略不当导致云计算费用爆炸

### 数据敏感度分级

- **极度敏感**：生产环境凭证（SSH 密钥/API Key/数据库密码）、安全漏洞详情、渗透测试结果
- **机密**：系统架构拓扑、IP 地址段、容量规划数据、事件复盘报告、安全策略
- **内部**：聚合性能指标、部署历史、公开的监控仪表盘
- **日志数据**：可能包含 PII（需脱敏处理），受留存策略和审计要求约束

### 监管/合规要求

- **中国**：《网络安全法》等级保护（等保 2.0）要求、《关键信息基础设施安全保护条例》、《数据安全法》数据分级分类
- **国际标准**：ISO 27001（信息安全管理）、SOC 2 Type II（服务组织控制）、PCI-DSS（涉及支付系统）
- **行业标准**：ITIL 4（IT 服务管理框架）、SRE 最佳实践（错误预算/SLO 管理）
- **变更管理**：SOX 合规要求变更审批记录（涉及财务系统）；CAB（变更顾问委员会）流程
- **安全合规**：NIST Cybersecurity Framework、MITRE ATT&CK 框架对齐、安全事件报告义务

### 性能/延迟要求

- **告警响应**：从告警触发到 Agent 开始分析 <30s，自动修复执行 <2min
- **监控采集**：指标采集间隔 15-60s，日志近实时（<10s 延迟）
- **部署流水线**：CI 构建+测试 <15min，金丝雀发布观测窗口可配置
- **安全检测**：实时入侵检测 <1s，漏洞扫描每日/每周
- **系统可用性**：监控/告警系统本身 99.99%

### HITL 要求

- **强制**：生产环境高风险变更（数据库迁移/核心服务部署/网络配置变更）需 CAB 审批；安全事件响应中的数据隔离/取证决策；自动修复策略上线需 SRE 审核；容量规划涉及预算的采购决策
- **建议**：非紧急自动修复可由 Agent 执行后人工复审；告警规则调整和基线更新定期审核
- **On-Call 协作**：Agent 作为"第一响应者"收集诊断信息和尝试已知修复，人工 On-Call 工程师作为升级路径和最终决策者

### 评估指标

- 可靠性：SLO 达成率、错误预算消耗率、MTTR（平均恢复时间）、MTTD（平均检测时间）
- 事件管理：事件数量趋势、自动修复成功率、根因分析准确率、升级率
- 部署：部署频率、部署失败率、回滚率、变更前置时间
- 效率：告警噪声比（可操作告警/总告警）、On-Call 工程师负载
- 成本：基础设施成本趋势、资源利用率、成本 vs 预算偏差

### 常见故障与恢复

| 故障模式                   | 恢复策略                                                                         |
| -------------------------- | -------------------------------------------------------------------------------- |
| Agent 自动修复引发级联故障 | 影响范围限制（blast radius）、自动修复操作需可逆、单次修复仅影响 N% 实例、熔断器 |
| 告警风暴淹没事件响应       | 告警聚合和去重、拓扑感知关联（上游故障抑制下游告警）、动态抑制规则               |

| 安全事件中凭证泄露         | 自动凭证轮换、密钥管理系统（Vault/KMS）、泄露检测后即时吊销和重签                |

### 领域风险画像

- **品牌危机应对失当**：Agent 在舆情危机时发布不当内容（定时发布未及时暂停）
- **内容调性偏差**：AI 生成内容与品牌调性（Tone of Voice）不一致，损害品牌形象
- **虚假营销**：生成内容含夸大宣传、虚假数据或未经授权使用名人肖像/品牌
- **合规风险**：违反《广告法》绝对化用语禁令、未标注广告标识、违规使用用户数据定向
- **ROI 归因失真**：错误归因模型导致预算错配，高估/低估渠道贡献

### 数据敏感度分级

- **PII（高）**：客户联系信息、行为画像数据、CRM 中的交易记录和偏好
- **机密**：品牌策略文档、未公开产品上市计划、营销预算明细、竞品分析报告
- **内部**：发布内容日历、A/B 测试方案、聚合营销指标

### 监管/合规要求

- **中国**：《广告法》（绝对化用语禁止、医疗/教育/金融广告特殊规定）、《互联网广告管理办法》（广告标识要求）、《反不正当竞争法》、《个人信息保护法》
- **美国**：FTC 广告真实性和代言指南（#ad 标识）、CAN-SPAM Act（邮件营销）、CCPA
- **欧盟**：不公平商业行为指令、ePrivacy 指令（Cookie 同意）、GDPR 营销同意、DSA 平台广告透明度
- **平台政策**：各社交平台广告和内容发布规范、KOL/网红合作的广告标识要求

### 性能/延迟要求

- **舆情监测**：负面舆情检测 <15min（品牌危机黄金响应时间），常规监测每小时更新
- **社交媒体发布**：内容生成 <5min/条（文案），图片/视频小时级
- **SEO 分析**：排名追踪每日更新，技术 SEO 审计每周
- **Campaign 报告**：准实时仪表盘（<15min 延迟）
- **客户分群**：批量更新每日，Campaign 触发式分群 <5min

### HITL 要求

- **强制**：品牌传播内容上线前需品牌团队审批；营销预算分配和调整需营销总监审批；品牌危机响应策略需公关团队制定；涉及法律风险的内容需法务审核
- **建议**：常规社交媒体帖子可由 Agent 生成后批量审批；A/B 测试方案启动前审核
- **危机管理**：检测到舆情危机时，Agent 立即暂停所有自动发布，通知公关团队接管

### 评估指标

- 品牌指标：品牌知名度、品牌好感度、SOV（Share of Voice）、NPS
- 营销效果：ROAS、CPA/CPL（获客/获线索成本）、转化率、LTV/CAC 比率
- 内容表现：互动率（点赞/评论/分享）、触达量、完播率（视频）、CTR
- SEO/SEM：自然搜索排名、有机流量增长、关键词覆盖率
- 社交媒体：粉丝增长率、互动率、品牌提及量、情感正负比

### 常见故障与恢复

| 故障模式                     | 恢复策略                                                             |
| ---------------------------- | -------------------------------------------------------------------- |
| 舆情危机期间自动发布不当内容 | 危机检测触发全渠道自动暂停发布、紧急通知公关团队、预案化响应模板     |
| AI 生成内容违反《广告法》    | 发布前合规检查 Agent（绝对化用语/虚假声明扫描）、法务审核工作流      |
| SEO 策略导致搜索引擎惩罚     | 白帽 SEO 护栏（禁止关键词堆砌/隐藏文本）、排名异常监控、惩罚恢复流程 |
| 营销数据归因模型失真         | 多归因模型对比验证、增量测试（Lift Test）作为校准基准                |
| 跨平台内容发布失败           | 发布队列重试机制、平台规格自适应转换、多平台发布状态监控仪表盘       |

### 通用平台能力要求

| 能力          | 要求                                                                                     |

| **多租户**    | 域隔离+共享基础设施。每个域有独立数据边界、工具权限和风控策略                            |
| **HITL 框架** | 按域/操作/风险等级可配置的审批工作流。支持同步（阻塞）和异步（排队）人工审核             |

| **工具注册**  | 可插拔工具集成层。每个域注册其工具并附 Schema、认证、速率限制和风险分类                  |
| **可观测性**  | Agent 工作流分布式追踪、LLM 调用日志、成本追踪、延迟监控、错误告警                       |

### DomainRecipe 原型扩展建议

现有4种原型不足以覆盖24个域的工作流模式。建议扩展为12种：

| 原型                           | 描述                          | 典型域                                       |
| ------------------------------ | ----------------------------- | -------------------------------------------- |
| CRUD-heavy（现有）             | 记录创建/读取/更新/删除为主   | 企业知识库、用户运营、人力资源               |
| Analytics（现有）              | 数据分析→洞察→建议            | 行业调研、用户运营、市场营销                 |
| Creative（现有）               | 内容生成→审核→发布            | 广告推广、广告素材制作、市场营销             |
| Realtime（现有）               | 事件驱动低延迟响应            | 量化交易、电商（风控）、在线直播             |
| **Trading（v3.0）**            | 信号→风控→执行→结算           | 量化交易、金融服务                           |
| **Compliance（v3.0）**         | 监控→检测→评估→报告           | 金融服务、财务、法务、医疗健康               |
| **Research（v3.0）**           | 收集→分析→综合→发表           | 行业调研、学术调研                           |
| **Adversarial（v3.0）**        | 攻击面分析→防御→审计→修复     | 代码开发（安全）、法务（诉讼）               |
| **Moderation（v3.1新增）**     | 内容摄入→多模态检测→处置→申诉 | 内容审核与安全、在线直播（审核链路）         |
| **Logistics（v3.1新增）**      | 预测→优化→调度→追踪→异常处理  | 供应链与物流、游戏上架（发行调度）           |
| **Conversational（v3.1新增）** | 意图识别→知识检索→回答→反馈   | 客户服务、教育培训（辅导）、医疗健康（分诊） |
| **IncidentOps（v3.1新增）**    | 告警→诊断→修复→复盘→预防      | IT 运维 SRE/DevOps                           |
