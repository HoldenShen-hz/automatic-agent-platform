# Migration

`migration/` answers two questions only: how to migrate and what to migrate.

## File Order

1. [00-migration-guideline.md](./00-migration-guideline.md)
   Migration principles, tiers, absorption methods, and judgment criteria.
2. [01-migration-scope.md](./01-migration-scope.md)
   Formal migration boundaries and exclusions for the new platform.

## Usage Principles

- First apply guideline judgment, then execute by scope.
- When conflicting with the platform skeleton, `architecture/00-platform-architecture.md` takes precedence.
