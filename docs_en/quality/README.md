# Quality

`quality/` consolidates testing and release quality documents, preventing them from being scattered across the root directory.

## File Order

1. [00-full-coverage-test-manual.md](./00-full-coverage-test-manual.md)
   Full coverage testing methodology and testing system documentation.
2. [01-release-checklist.md](./01-release-checklist.md)
   Checklist items that must be satisfied before production release.

## Usage Guidelines

- When designing a verification plan, refer to `00` first.
- When conducting release sign-off, refer to `01`.
