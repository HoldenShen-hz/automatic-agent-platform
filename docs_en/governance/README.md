# Governance

> `governance/` records cross-document, cross-module, and cross-team governance rules.

## Current Files

- [source_of_truth.md](./source_of_truth.md)
- [change_control.md](./change_control.md)
- [naming_and_directory_conventions.md](./naming_and_directory_conventions.md)
- [glossary_and_terminology.md](./glossary_and_terminology.md)

## Uses

- Define source of truth rules.
- Define document update and conflict handling rules.
- Define cross-module boundaries, terminology, directory, and change governance methods.
- Define unified writing for canonical id, business alias, and naming formats.
- Define unified meanings of high-frequency professional terms for core objects, states, events, governance, security, storage, and operations.

## Writing Rules

- Governance is a long-term rule layer, does not write temporary execution items.
- If governance rules affect platform behavior, should link corresponding ADR or contract.
- Same rule only keeps one master version, avoid multiple places branching.
