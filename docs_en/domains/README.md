## Domain Model Structure

Each domain may contain:
- `domain.yaml` - Domain definition and configuration
- `workflows/` - Domain-specific workflow definitions
- `roles/` - Domain-specific role definitions
- `plugins/` - Domain-specific plugin configurations

## Usage

- Domains are business capability boundaries within the platform.
- Each domain can have its own workflows, roles, and tools.
- Domains are registered through the domain registry system.
