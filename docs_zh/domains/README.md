# Domains 索引

`docs_zh/domains/` 保存各业务域的中文说明文档。每个域目录可包含领域模型、工作流定义、角色说明和插件配置。

## 目录说明

- `academic-research/` - 学术研究
- `advertising/` - 广告投放
- `coding/` - 软件开发
- `content-moderation/` - 内容审核
- `creative-production/` - 创意制作
- `customer-service/` - 客户服务
- `data-engineering/` - 数据工程
- `ecommerce/` - 电商
- `education/` - 教育培训
- `finance-accounting/` - 财务与会计
- `financial-services/` - 金融服务
- `game-dev/` - 游戏开发
- `game-publishing/` - 游戏上架
- `healthcare/` - 医疗健康
- `human-resources/` - 人力资源
- `industry-research/` - 行业研究
- `it-operations/` - IT 运维
- `knowledge-base/` - 企业知识库
- `legal/` - 法务
- `live-streaming/` - 在线直播
- `marketing/` - 市场营销
- `quant-trading/` - 量化交易
- `supply-chain/` - 供应链
- `user-operations/` - 用户运营

## 领域文档结构

每个业务域目录通常包含：

- `domain.yaml` - 领域定义与配置
- `workflows/` - 领域工作流
- `roles/` - 领域角色定义
- `plugins/` - 领域插件配置

## 使用规则

- 业务域是平台中的能力边界。
- 每个业务域可以拥有自己的工作流、角色和工具。
- 领域接入与注册以 domain registry 主链为准。
