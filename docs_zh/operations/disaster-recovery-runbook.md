# 灾难恢复操作手册

本文档补充 ADR-031 的执行步骤，用于区域不可用、数据库不可写、队列不可用和关键运行时恢复。

## 触发条件

- 主区域 API、worker 或数据库连续不可用超过 RTO 门限。
- 数据写入失败且无法通过局部重试恢复。
- 队列或事件证据链出现不可接受的延迟或丢失风险。

## 准备检查

1. 确认 incident commander 和审批人。
2. 冻结非必要部署。
3. 导出当前健康检查、队列深度、数据库复制状态和最近事件证据。
4. 确认备份快照、恢复点和目标区域容量。

## 恢复步骤

1. 切换入口流量到备用区域或降级入口。
2. 启动备用 worker 和控制面服务。
3. 恢复数据库到目标恢复点，并执行只读校验。
4. 恢复队列消费，先启用低并发，再逐步恢复正常并发。
5. 执行 runtime recovery 定向检查，确认 stale、blocked、dead-letter 视图可用。
6. 记录恢复窗口、数据缺口和人工审批动作。

## 回滚

- 若备用区域恢复失败，停止新增写入并保持只读状态。
- 回退流量前必须确认主区域写入路径和事件证据链一致。

## 证据

- 健康检查输出。
- 数据库恢复日志。
- 队列深度和消费恢复记录。
- runtime recovery 报告。
- 事后 postmortem。
