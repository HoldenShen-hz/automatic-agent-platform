# 安全入职与离职流程

本文档定义人员、服务账号和自动化凭据的访问生命周期，覆盖入职授权、权限变更、离职回收和审计留痕。

## 入职

- 由直属负责人提交访问申请，说明角色、业务范围、所需环境和有效期。
- 生产权限默认拒绝，必须说明操作场景并绑定审批人。
- 初始权限按最小权限授予，优先使用组和角色，不直接给个人绑定长期特权。
- 所有 secret 通过批准的 secret manager 或部署环境注入，禁止写入 `.env`、脚本参数、日志和 fixture。

## 权限变更

- 权限升级必须记录原因、期限和回滚计划。
- 临时权限到期自动回收；无法自动回收时必须建立日历提醒和审计记录。
- 跨团队访问需要资源 owner 和安全 owner 双重确认。

## 离职和转岗

- 离职当天撤销 SSO、VPN、代码仓库、CI/CD、云账号、数据库和 observability 后台访问。
- 轮换个人曾接触的共享 token、机器人 token 和长期 API key。
- 检查最近 30 天审计日志，确认无异常导出、权限提升或失败登录爆发。

## 证据

- 访问申请单。
- 审批记录。
- 权限变更 diff。
- secret 轮换记录。
- 离职回收清单。
