# Guides 索引

本目录存放操作性内容、写作规范和落地流程。

## 指南列表

| 文件 | 标题 | 简介 |
|------|------|------|
| `contributing.md` | 贡献指南 | 文档拆分后的维护分工、写作规则、迁移规则和提交流程 |
| `quickstart.md` | 快速入门 | 推荐阅读顺序、当前建议实现范围、Phase 落地顺序建议、故障排查 |
| `division-authoring.md` | 事业部创作指南 | 如何创建和配置新的业务事业部（Division） |
| `skill-authoring.md` | 技能创作指南 | 如何开发和注册新的 Agent 技能（Skill） |

## 文档层次

| 类型 | 职责 | 位置 |
|------|------|------|
| **总纲** | 平台是什么 | `docs_zh/architecture/00-*.md` |
| **ADR** | 为什么这么设计 | `docs_zh/adr/*.md` |
| **Guide** | 具体怎么做 | `docs_zh/guides/*.md` |

## 更新规则

- 新的架构决策进入对应 ADR
- 影响全局认知的变化，同步更新总纲中的摘要和导航
- 事业部写法、开发流程、维护约束等内容进入 guides
- 需要追踪来源时，在文件底部补充"来源章节"
- 发现原始大文档与拆分文档冲突时，以拆分文档为后续维护目标
