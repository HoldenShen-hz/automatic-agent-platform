# ADR-032 部署架构

- 状态：Accepted
- 决策日期：2026-04-03

## 背景

平台需要支持多环境部署（dev/test/staging/pre-prod/prod），并支持不同规模的部署形态。

## 决策

### 部署阶段

| 阶段 | 说明 |
|------|------|
| D1 单体 | ≤10 并发，当前默认配置 |
| D2 多进程 | 10-100 并发 |
| D3 分布式 | 100-1000 并发 |
| D4 K8s 集群 | 5000+ 并发 |

### 5 个环境

| 环境 | 用途 |
|------|------|
| dev | 开发环境 |
| test | 单元/集成测试 |
| staging | 预发布测试 |
| pre-prod | 生产前验证 |
| prod | 生产环境 |

### Worker 池隔离

- worker-pool/ 支持能力类别隔离
- 不同类型任务使用不同 Worker 池
- 防止资源争抢

### 部署方式

- Helm values 管理 K8s 配置
- Terraform tfvars 管理基础设施配置

## 后果

优点：

- 多环境分离便于测试和发布
- Worker 池隔离提高稳定性
- IaC 方式便于环境一致性

代价：

- 多环境增加运维复杂度
- 部署流程需要标准化
