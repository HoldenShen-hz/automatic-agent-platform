# ADR-044 非技术用户体验架构

- 状态：Accepted
- 决策日期：2026-04-20

## 背景

非技术用户（业务操作者、运营人员）需要简化的交互方式，不需要理解底层 API 和技术细节。

## 决策

### 用户角色

| 角色 | 说明 | 界面 |
|------|------|------|
| business_operator | 业务操作者 | 简化界面 |
| org_node_owner(team) | `OrgNodeType.team` 的团队负责人 | 管理视图 |
| executive | 高管 | 报表视图 |
| admin | 管理员 | 全功能界面 |

### 简化交互模式

- 模板化任务创建
- 自然语言输入
- 可视化进度跟踪
- 一键审批/拒绝

### 通知与反馈

- 推送通知
- 邮件通知
- 站内信
- 状态变更实时更新

### 辅助功能

| 功能 | 说明 |
|------|------|
| 任务模板 | 常见任务预定义模板 |
| 快捷操作 | 常用操作一键执行 |
| 历史记录 | 操作历史可追溯 |
| 导出报表 | 数据导出支持 |

## 后果

优点：

- 降低非技术用户使用门槛
- 模板化提高效率
- 实时反馈提升体验

代价：

- 多角色界面增加复杂度
- 模板维护需要持续投入
