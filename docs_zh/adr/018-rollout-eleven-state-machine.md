# ADR-018 Rollout 十一态状态机与六阶段发布

- 状态：Superseded by ADR-075
- 决策日期：2026-04-17
- 当前权威规范：ADR-075（六级发布状态机）

## 说明

> **Historical record only. Do not implement from this document.**
> 自 2026-04-17 起，ADR-075 是唯一权威的 rollout / release 状态机规范。

本文档仅保留为历史决策记录，用来说明平台曾经讨论过一版更早的 rollout 设计。以下内容不得再作为实现、测试、评审或契约依据：

- 早期的十一态 `RolloutStatus` 设计草案
- 早期的五级发布分层命名
- 早期的自动回滚阈值与门限描述

## 历史摘要

在 ADR-075 落地之前，平台曾评估过一版更细粒度的 rollout 状态机，希望覆盖：

- shadow 到 canary / staged / stable 的渐进发布演进
- 自动回滚与暂停恢复
- rollout 历史轨迹留存

这些历史讨论已经全部被 ADR-075 取代。若需要当前有效的状态集合、级别定义、回滚条件和实现契约，请直接查阅 ADR-075 及其配套实现。

## 迁移结论

- 不再维护 ADR-018 中的状态枚举、级别表和阈值规则。
- 新实现与新测试只能引用 ADR-075。
- 若发现代码、文档或审计仍引用 ADR-018 的具体规则，应视为漂移并迁移到 ADR-075。
