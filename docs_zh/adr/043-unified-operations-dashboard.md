# ADR-043 Unified Operations Dashboard

- 状态：Accepted
- 决策日期：2026-04-20

## 背景

运维人员需要一个统一的视图来了解平台整体状态，而不是分散在多个监控系统中。

## 决策

### Dashboard 架构

| 模块 | 说明 |
|------|------|
| 系统概览 | 平台整体健康状态 |
| 任务看板 | 实时任务状态和进度 |
| 事件中心 | 告警和事件列表 |
| 成本报表 | 资源消耗和成本 |
| 性能监控 | P99/延迟/错误率 |
| 安全态势 | 认证/授权/审计 |

### WebSocket 实时推送

- `/ws/v1/stream` 端点
- DashboardWebSocketServer 实现
- 实时更新状态

### 视图分层

| 层级 | 受众 | 内容 |
|------|------|------|
| 管理层 | 高管 | 业务指标、健康状态 |
| 运维层 | SRE | 系统指标、告警、容量 |
| 开发层 | 研发 | 任务详情、日志、trace |

### 告警聚合

- 减少告警疲劳
- 智能分组
- 根因分析辅助

## 后果

优点：

- 统一视图提高运维效率
- 实时推送保证及时响应
- 分层视图满足不同受众

代价：

- 仪表盘维护成本
- 实时数据管道复杂度
